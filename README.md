# 選挙掲示板管理アプリ

選挙掲示板の設置場所を **投票区ごと** にチェックリスト形式で管理するWebアプリです。
Googleスプレッドシートをデータ元として、現地確認が終わった項目にチェックを入れると、
チェック状態とチェック日時がスプレッドシートに保存されます。
複数端末から見ても同じ進捗を確認できます。

> 無料運用を優先するため Google Maps API は使用しません。地図表示・ピン表示は行わず、
> 各行から「地図」リンクでGoogleマップ検索を新しいタブで開く方式です。

## 構成

```
Reactアプリ
   ↓
Google Apps Script Web API
   ↓
Googleスプレッドシート
```

## スプレッドシートの列構成

| 列 | 内容 |
| -- | -- |
| A列 | 投票区 |
| B列 | 住所 |
| C列 | 施設名 |
| D列 | チェック状態 (TRUE / FALSE) |
| E列 | チェック日時 |

- 1行目は **見出し行** として扱い、データは2行目以降を読み込みます。
- 記入者列・F列は使用しません。

## セットアップ

### 1. Google Apps Script の準備

1. 対象のGoogleスプレッドシートを開く。
2. メニュー **拡張機能 > Apps Script** を開く。
3. エディタの内容を [`gas/Code.gs`](gas/Code.gs) の中身に置き換えて保存する。
4. **デプロイ > 新しいデプロイ** を選ぶ。
   - 種類: **ウェブアプリ**
   - 実行するユーザー: **自分**
   - アクセスできるユーザー: **全員**
5. 発行された **ウェブアプリのURL**（`https://script.google.com/macros/s/.../exec`）を控える。

> 初回デプロイ時にアクセス権限の承認を求められます。承認して進めてください。
> コードを修正した場合は **デプロイ > デプロイを管理 > 編集 > バージョン: 新バージョン** で再デプロイします。

### 2. 環境変数の設定

`.env.example` をコピーして `.env` を作成し、上で控えたURLを設定します。

```
VITE_GAS_API_URL=https://script.google.com/macros/s/xxxxxxxx/exec
```

`.env` は `.gitignore` に含まれているためコミットされません。

### 3. 依存パッケージのインストール

```bash
npm install
```

### 4. 開発サーバーの起動

```bash
npm run dev
```

表示されたローカルURL（例: `http://localhost:5173`）をブラウザで開きます。

### 5. 本番ビルド

```bash
npm run build      # dist/ に出力
npm run preview    # ビルド結果をローカルで確認
```

`dist/` を任意の静的ホスティング（GitHub Pages / Netlify / Cloudflare Pages など無料枠）に配置すれば公開できます。

## 動作確認の手順

1. スプレッドシートのA〜C列に「投票区・住所・施設名」を入力しておく。
2. `npm run dev` でアプリを開く。
3. 投票区ごとに見出しが表示され、その下に各行が
   **チェックボックス → 施設名 → 住所** の順で並ぶことを確認する。
4. チェックボックスをONにすると、
   - 行が薄いグレーになり、施設名・住所に取り消し線が付き、「済」ラベルが表示される。
   - スプレッドシートの **D列がTRUE**、**E列に現在日時** が入る。
5. チェックを外すと、
   - 通常表示に戻る。
   - スプレッドシートの **D列がFALSE**、**E列が空欄** になる。
6. 別の端末・ブラウザで開き直しても同じチェック状態が表示されることを確認する。
7. 各行の「地図」リンクを押すと、新しいタブでGoogleマップ検索が開くことを確認する。

## 機能一覧

- 投票区(A列)ごとのグルーピング表示
- `チェックボックス → 施設名 → 住所` の行レイアウト
- チェック済みの視覚表示（グレー背景・取り消し線・「済」ラベル・確認日時）
- チェック状態をGoogleスプレッドシートへ保存（複数端末で進捗共有）
- Googleマップ検索リンク（`https://www.google.com/maps/search/?api=1&query=施設名+住所`）
- スマホ向けレスポンシブ対応（大きめのチェックボックス・余裕のある行高さ・住所の折り返し・縦スクロール）

## 技術スタック

- React 18 + Vite
- Google Apps Script (ウェブAPI)
- Googleスプレッドシート
- CSS（レスポンシブ）

## 注意点

- 進捗の保存先はGoogleスプレッドシートです。localStorageはメインの進捗保存に使用していません。
- Google Maps API は使用していません（地図埋め込み・ピン表示・ピン削除は未実装）。
- 表記は「選挙区」ではなく「投票区」で統一しています。
